# إصلاح زر عرض الخريطة التفاعلية ✅

## المشكلة
عند الضغط على زر "عرض الخريطة التفاعلية" لا يحدث شيء.

## السبب الجذري
المكون `MapViewButton` في mostshfa_new كان مختلفاً تماماً عن النسخة الأصلية في mostshfa_pro:
- النسخة الخاطئة: كانت تتوقع props مختلفة (`isMapView`, `onToggle`) وتعمل كزر toggle بسيط
- النسخة الصحيحة: تفتح modal يحتوي على خريطة تفاعلية كاملة باستخدام Leaflet

## الحل

### 1. تحديث MapViewButton.tsx
استبدال المكون بالكامل ليطابق النسخة الأصلية:
- إضافة state لإدارة ظهور/إخفاء الخريطة
- استخدام AnimatePresence لعرض modal الخريطة
- تمرير المستشفيات إلى مكون InteractiveMap

### 2. إنشاء InteractiveMap.tsx
مكون جديد يعرض خريطة تفاعلية كاملة:
- استخدام Leaflet و react-leaflet
- Dynamic import لتجنب مشاكل SSR
- عرض المستشفيات كعلامات على الخريطة
- بحث مباشر على الخريطة
- popup لكل مستشفى مع رابط للتفاصيل

### 3. تثبيت المكتبات المطلوبة
```bash
pnpm add leaflet react-leaflet @types/leaflet
```

## ملاحظات مهمة

### إحداثيات المستشفيات
حالياً يتم توليد إحداثيات عشوائية للعرض التوضيحي:
```typescript
const lat = 26 + Math.random() * 5;
const lng = 30 + Math.random() * 5;
```

**للإنتاج**: يجب إضافة حقول `latitude` و `longitude` إلى جدول المستشفيات في قاعدة البيانات واستخدامها بدلاً من الإحداثيات العشوائية.

### تحذيرات peer dependencies
ظهرت تحذيرات حول React 19 vs 18، لكنها لا تؤثر على عمل المكتبة:
```
react-leaflet 5.0.0 expects react@^19.0.0 but found 18.3.1
```
هذا طبيعي ولا يسبب مشاكل في التشغيل.
