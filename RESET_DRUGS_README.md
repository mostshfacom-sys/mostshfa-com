# إعادة تعيين قاعدة بيانات الأدوية

هذا الدليل يوضح كيفية مسح جميع بيانات الأدوية من قاعدة البيانات مع الاحتفاظ بالجداول وبنية قاعدة البيانات.

## ⚠️ تحذير مهم

- **سيتم مسح جميع بيانات الأدوية نهائياً**
- **سيتم الاحتفاظ بالجداول وبنية قاعدة البيانات**
- **قاعدة البيانات المصدر لن تتأثر**

## 🛠️ الأدوات المتاحة

### 1. تشغيل سريع (مستحسن)

#### Windows PowerShell:
```powershell
.\reset-drugs-database.ps1
```

#### Windows CMD:
```cmd
reset-drugs-database.bat
```

### 2. تشغيل يدوي

#### مسح البيانات فقط:
```bash
node scripts/clear-drugs-data.js
```

#### التحقق من الحالة فقط:
```bash
node scripts/verify-drugs-cleared.js
```

#### العملية الكاملة:
```bash
node scripts/reset-drugs-database.js
```

## 📋 ما يحدث أثناء العملية

### المرحلة 1: مسح البيانات
1. عرض الإحصائيات الحالية
2. مسح جميع الأدوية
3. مسح جميع أصناف الأدوية
4. إعادة تعيين معرفات الجداول

### المرحلة 2: التحقق
1. فحص عدد الأدوية والأصناف
2. التأكد من وجود الجداول
3. فحص الفهارس
4. اختبار إدراج بيانات تجريبية
5. عرض ملخص الحالة

## 🗂️ الجداول المتأثرة

- `drugs` - جدول الأدوية
- `drug_categories` - جدول أصناف الأدوية

## 🔒 الجداول غير المتأثرة

جميع الجداول الأخرى ستبقى كما هي:
- المستشفيات والعيادات
- المختبرات والصيدليات
- المقالات والمستخدمين
- جميع البيانات الأخرى

## 📊 مثال على النتيجة

```
📊 الإحصائيات الحالية:
   - عدد الأدوية: 28884
   - عدد أصناف الأدوية: 156

🗑️ مسح جميع الأدوية...
   ✅ تم مسح 28884 دواء

🗑️ مسح جميع أصناف الأدوية...
   ✅ تم مسح 156 صنف

📊 الإحصائيات النهائية:
   - عدد الأدوية: 0
   - عدد أصناف الأدوية: 0

✅ تم مسح جميع بيانات الأدوية بنجاح!
```

## 🚀 الخطوات التالية

بعد مسح البيانات، يمكنك:

1. **إضافة بيانات جديدة** باستخدام scripts الاستيراد
2. **إنشاء أصناف جديدة** من لوحة الإدارة
3. **استيراد من مصدر آخر** باستخدام API

## 🛡️ نصائح الأمان

1. **إنشاء نسخة احتياطية** قبل تشغيل العملية
2. **التأكد من البيئة** (تطوير وليس إنتاج)
3. **اختبار العملية** على بيانات تجريبية أولاً

## 📞 الدعم

إذا واجهت أي مشاكل:
1. تحقق من logs العملية
2. تأكد من إعدادات قاعدة البيانات
